## 📦 Package Management Migration (pip → uv)

### Why uv?
- **Speed**: 10-100x faster dependency resolution
- **Reliability**: Lockfiles pin ALL transitive dependencies
- **Developer Experience**: Single command (`uv sync`) vs pip complexity
- **Modern**: Built with Rust, actively maintained

### Changes Made

#### 1. Created pyproject.toml for Each Service
**Location:** `services/{service}/pyproject.toml`

**Auth Service Example:**
```toml
[project]
name = "auth-service"
version = "0.1.0"
requires-python = ">=3.11"
dependencies = [
    "Django==4.2.17",
    "djangorestframework==3.15.2",
    "djangorestframework-simplejwt==5.4.0",
    "psycopg2-binary==2.9.9",
    "django-cors-headers==4.6.0",
    "uvicorn[standard]==0.34.0",
]
```

**Key Decision:** Removed `[build-system]` section because Django apps aren't packages to be installed.

#### 2. Generated uv.lock Files
**Command Used:** `uv lock` (in each service directory)

**Result:** 15KB lockfiles per service pinning all transitive dependencies.

#### 3. Updated Dockerfiles
**Before:**
```dockerfile
RUN pip install -r requirements.txt
CMD ["python", "manage.py", "runserver", "0.0.0.0:8001"]
```

**After:**
```dockerfile
RUN pip install uv
COPY pyproject.toml uv.lock ./
RUN uv sync --frozen --no-dev
CMD ["uv", "run", "uvicorn", "core.asgi:application", "--host", "0.0.0.0", "--port", "8001", "--reload"]
```

**Benefits:**
- Fast, reproducible builds
- Frozen lockfile prevents unexpected updates
- `uv run` handles virtualenv automatically
- `--reload` enables hot-reload

#### 4. Created ASGI Applications
**Location:** `services/{service}/core/asgi.py`

**Content:**
```python
import os
from django.core.asgi import get_asgi_application

os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'core.settings')
application = get_asgi_application()
```

**Why Required:** Uvicorn requires ASGI interface (Django 4.2+ supports this).

### Issues Encountered & Resolved

| Issue | Solution |
|-------|----------|
| `ValueError: Unable to determine which files to ship inside the wheel` | Removed `[build-system]` from pyproject.toml |
| `uv.lock file not found with --frozen` | Generated lockfiles locally first with `uv lock` |
| `.venv/bin/python not found` in containers | Used `uv run` instead of direct venv path |
| Missing TEMPLATES configuration | Added TEMPLATES setting to all services' settings.py |
| `psycopg2-binary==2.9` incompatible | Updated to `psycopg2-binary==2.9.9` |

## 🔄 Uvicorn + Hot-Reload Implementation

### Why Uvicorn?
- **ASGI Support**: Django 4.2+ native ASGI support
- **Performance**: Faster than Django's development server
- **Hot-Reload**: Instant code changes without restart
- **Production-Ready**: Can be used in production (unlike runserver)

### Implementation
```dockerfile
# In Dockerfile
CMD ["uv", "run", "uvicorn", "core.asgi:application", "--host", "0.0.0.0", "--port", "8001", "--reload"]
```

### Hot-Reload Behavior
- **Monitored Directories**: `/app` (volume mount)
- **Reload Trigger**: Any .py file change
- **Delay**: ~2-5 seconds
- **No Restart Required**: Changes auto-apply

### Volume Mounts (Critical for Hot-Reload)
```yaml
# In docker-compose.yml
volumes:
  - ./services/auth:/app  # Maps host code to container
```

**Result:** Edit code on host → uvicorn detects change → auto-reloads

## 🚚 Traefik Integration (Replacing nginx)

### Why Traefik?
- **Zero Configuration**: Auto-discovers services from Docker labels
- **Dynamic Routing**: No config reload when adding services
- **Built-in Dashboard**: Real-time monitoring at http://localhost:8080
- **SSL/TLS**: Automatic Let's Encrypt certificates (production)
- **Health Checks**: Built-in service health monitoring

### Architecture
```
Internet (Port 80/443)
    ↓
Traefik (reverse proxy)
    ↓ (auto-discovers via Docker labels)
├── auth (port 8001) - Label: PathPrefix(`/api/auth`)
├── playlist (port 8002) - Label: PathPrefix(`/api/playlists`)
├── track (port 8003) - Label: PathPrefix(`/api/tracks`)
├── search (port 8004) - Label: PathPrefix(`/api/search`)
└── collaboration (port 8005) - Label: PathPrefix(`/api/collab`)
```

### Docker Compose Configuration
```yaml
traefik:
  image: traefik:latest
  command:
    - "--api.dashboard=true"
    - "--providers.docker=true"
    - "--providers.docker.exposedbydefault=false"
    - "--providers.docker.network=spotify-collab_default"
    - "--entrypoints.web.address=:80"
    - "--api.insecure=true"
    - "--accesslog=true"
  ports:
    - "80:80"      # HTTP gateway
    - "443:443"    # HTTPS gateway
    - "8080:8080"  # Dashboard
  volumes:
    - /var/run/docker.sock:/var/run/docker.sock:ro

auth:
  labels:
    - "traefik.enable=true"
    - "traefik.http.routers.auth.rule=PathPrefix(`/api/auth`)"
    - "traefik.http.services.auth.loadbalancer.server.port=8001"
```

### Key Features Implemented
- ✅ Service auto-discovery from Docker labels
- ✅ PathPrefix-based routing
- ✅ Load balancer configuration
- ✅ Access logging enabled
- ✅ Dashboard exposed on localhost:8080
- ✅ Network isolation for security

### Adding New Services (Zero Config)
```yaml
new-service:
  labels:
    - "traefik.enable=true"
    - "traefik.http.routers.new-service.rule=PathPrefix(`/api/new-service`)"
    - "traefik.http.services.new-service.loadbalancer.server.port=8006"
```

Traefik automatically discovers and routes to the new service!

## 🏥 Health Check Endpoints

### Purpose
- **Kubernetes**: Liveness/readiness probes
- **Load Balancers**: Health monitoring
- **Monitoring**: Automated service status checks
- **Development**: Quick service verification

### Implementation
**Added to:** All 5 services (auth, playlist, track, search, collaboration)

**Code Pattern:**
```python
@api_view(['GET'])
@permission_classes([permissions.AllowAny])
def health_check(request):
    """
    Health check endpoint for monitoring and orchestration
    Returns 200 if service and database are healthy
    """
    try:
        # Check database connection
        with connection.cursor() as cursor:
            cursor.execute('SELECT 1')
            cursor.fetchone()

        return Response({
            'status': 'healthy',
            'service': 'service-name',
            'database': 'connected'
        }, status=200)
    except Exception as e:
        return Response({
            'status': 'unhealthy',
            'service': 'service-name',
            'database': 'disconnected',
            'error': str(e)
        }, status=503)
```

### Endpoints
```
http://localhost/api/auth/health/
http://localhost/api/playlists/health/
http://localhost/api/tracks/health/
http://localhost/api/search/health/
http://localhost/api/collab/health/
```

### Healthy Response
```json
{
    "status": "healthy",
    "service": "auth",
    "database": "connected"
}
```

### Unhealthy Response
```json
{
    "status": "unhealthy",
    "service": "auth",
    "database": "disconnected",
    "error": "error details"
}
```
